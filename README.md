# wst — Terminal Sound Interface

## Цель

Создать **звуковой интерфейс для терминала**, позволяющий разработчику полноценно работать без визуального контроля экрана. Озвучивание вывода терминала, ввода и событий от приложений (vim, git, компиляторы и т.д.).

### Контекст

Пользователь — разработчик, владеющий слепым методом печати (RU/EN). Зрение сильно устаёт, поэтому нужна альтернатива визуальному мониторингу вывода терминала.

---

## Основные задачи

### 1. Перехват вывода терминала
- Захват stdout/stderr от команд
- Обработка ANSI escape-последовательностей (цвета, курсор, обновление экрана в реальном времени)
- Поддержка интерактивных программ (top, htop, vim, nano)

### 2. Озвучивание
- Кодирование символов в уникальные звуковые сигналы (частота + огибающая)
- Озвучка вывода команд (результат ls, git status, ошибки компиляции)
- Опционально: озвучка ввода (подтверждение набираемых команд)
- Звуковые маркеры событий (ошибка, успех, новая строка, конец вывода)

### 3. Интеграция с vim и другими TUI-программами
- Режим "vim": озвучка текущего положения курсора, режима (Normal/Insert/Visual), сообщений
- Режим "команда": быстрая озвучка результата выполнения
- Режим "поток": непрерывное озвучивание (tail -f, логи)

### 4. Настройка и кастомизация
- Регулировка скорости озвучки (задержка между символами)
- Фильтрация шума (игнорировать приглашения, повторяющийся вывод)
- Профили для разных задач (кодинг, git, навигация по файлам)
- Горячие клавиши для паузы/повтора последнего вывода

## Альтернативные подходы (для обсуждения)

### Вариант A: Обёртка над терминалом (текущий)
- `wsh` вместо `bash`/`zsh` — оболочка, которая перехватывает всё
- **Плюсы:** полный контроль, работает с любыми программами
- **Минусы:** сложная реализация, обработка TUI

### Вариант B: Интеграция с существующими screen reader
- Использование Orca (Linux), NVDA/JAWS (Windows)
- **Плюсы:** уже работает с vim/terminal
- **Минусы:** менее кастомизируемо, общий-purpose, не для разработчиков

### Вариант C: TTS вместо тональных сигналов
- Использовать синтез речи (RHVoice, festival, pyttsx3)
- **Плюсы:** понятнее для длинных сообщений
- **Минусы:** медленнее, не подходит для "быстрого сканирования"

### Вариант D: Гибрид
- Короткие ошибки/статусы → тональные сигналы
- Длинный вывод → TTS (синтез речи)
- **Рекомендуемый вариант**

## Технические вопросы

1. **Windows vs Linux:** В Windows нет pty в привычном смысле. Нужен подход через ConPTY (Windows 10+) или через хук в терминал.
2. **Vim:** Vim обновляет экран целиком через ANSI. Нужна дельта-обработка (сравнивать с предыдущим состоянием, озвучивать только изменения).
3. **Скорость:** Посимвольная озвучка медленная. Нужны режимы "быстро" (TTS) и "точно" (тональные сигналы).
4. **Звук:** `sounddevice` + `numpy` работают, но нужна более низкоуровневая очередь, чтобы не блокировать вывод.

## Использование

### Звуковой режим (основной)

```powershell
# Одна команда — и весь вывод озвучивается
python cli.py wsh

# Появится prompt:
# wsh> git status    ← озвучит результат
# wsh> dir           ← озвучит список файлов
# wsh> exit          ← выход
```

### Разовая команда

```powershell
python cli.py wsh git status    # запустить и озвучить
python cli.py wsh dir
python cli.py speak "текст"     # просто озвучить текст
```

### Настройка скорости

В `config.py` измени `DURATION_CHAR`:
- `0.5` — медленно, разборчиво (по умолчанию)
- `0.1` — нормально
- `0.05` — быстро
